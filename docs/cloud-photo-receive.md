# クラウド写真受信仕様

## 受信対象

- 現在参加中の`siteId`に属する写真だけをRLS経由で取得する。
- `photo_objects.status = complete`かつ`upload_completed_at`がある写真だけを端末の写真一覧へ反映する。
- `photoUid`が既存写真と一致する場合、SHA-256一致時だけ同一写真として扱う。aoPIC由来メタデータは上書きしない。
- SHA-256が異なる場合は同期全体を中止し、既存写真を維持する。

## 通信とキャッシュ

- 初期値は「サムネイルのみ」。一覧の表示付近にあるサムネイルだけを遅延取得する。
- 原寸は詳細表示で明示的に開いた時、またはWi-Fiと明確に判定できる時の台帳表示で取得する。
- 原寸一括取得は利用者の確認後に1枚ずつ実行する。モバイル通信・回線不明時も確認を必須とする。
- 取得したBlobは`aoALBDB/cloudFiles`へ`photoUid:kind`をキーとして保存し、同じ画像を再取得しない。
- キャッシュ削除は`cloudFiles`だけを対象とし、`photos`、`projects`、`ledgers`、クラウド原本を変更しない。

## 台帳と印刷

- クラウド写真はZIP写真と同じ`photos`ストアへ正規化し、`source`/`sources`と`cloud`参照情報で由来を保持する。
- 台帳配置は従来どおり写真の`internalId`を参照し、写真メタデータを更新しない。
- 原寸が端末になく、安全な回線で取得できない場合は対象枠を警告し、印刷を実行しない。

## オフライン

- 端末へ保存済みの写真・サムネイルはオフラインでも表示する。
- 未取得画像は空の画像枠と案内で示し、完成済みとして扱わない。
- Realtime再接続後または「新着写真を確認」で完成写真一覧を再取得する。イベントは通知として扱い、最終状態はDB再取得結果を正とする。
