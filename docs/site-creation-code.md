# 現場作成専用コードの管理

現場作成専用コードは、匿名認証端末が新しい共有現場を作成する時だけ使う会社共通コードです。現場へ参加するための参加コードとは別物です。

## 安全要件

- 平文はDB、Git、チャット、アプリ設定へ保存しません。
- ブラウザへservice role key、Secret key、DBパスワードを渡しません。
- アプリはコードを送信後、入力欄から消去します。
- DBには`pgcrypto.crypt()`のハッシュだけを保存します。
- コードの秘密性が主な防御です。匿名ユーザー単位の回数制限だけでは、アカウントを作り直す攻撃を完全には防げません。

## 初回登録と変更

1. Supabase Dashboardで対象プロジェクト名を確認します。
2. SQL Editorで`supabase/operations/202607220002_rotate_site_creation_code.sql`全体を実行します。
3. 結果に一度だけ表示されるコードを、会社の安全なパスワード管理場所へ保存します。
4. チャット、GitHub、メモ欄、画面キャプチャへ残しません。

SQL内へ平文コードを書き込まず、PostgreSQLが安全なランダムコードを生成します。再実行すると旧コードは直ちに無効になり、新コードだけが有効になります。

## 紛失・漏えい時

同じ変更SQLを再実行してください。新コードを安全に控えた後、旧コードが拒否されることを確認します。既存現場、参加コード、所属端末、写真には影響しません。

## 作成制限

- 誤った現場作成コードは、匿名ユーザーごとに15分間5回までです。
- 正しいコードでも、匿名ユーザー1人につき1時間3現場までです。
- 現場IDを変更しても誤入力回数はリセットされません。
- 将来CAPTCHA／Turnstileを追加できるよう、作成は`create_site` RPCへ集約しています。
